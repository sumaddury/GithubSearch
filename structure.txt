Key services
- Postgres: normalized issue/PR/comment metadata storage
- Meilisearch: lexical search index
- API: REST endpoints for indexing, querying, and viewing results
- Worker: GitHub ingestion + normalization + Meilisearch sync

Data flow
1) User submits repo + description in UI.
2) API checks index status and triggers or reports ingestion if needed.
3) Worker paginates GitHub REST API and stores metadata in Postgres.
4) Worker updates Meilisearch index from Postgres.
5) API queries Meilisearch and returns ranked results.
